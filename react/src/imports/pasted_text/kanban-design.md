🧠 PROMPT PARA FIGMA (KANBAN EN AGENDA)

Copia y pega esto 👇

🎯 PROMPT:

Diseña una vista Kanban para el módulo “Procesos y Tareas” dentro de la pestaña “Agenda” de un sistema ERP SaaS llamado Índice.

El Kanban debe ser una vista alternativa a la tabla de tareas, dentro de la misma pantalla (no una pantalla nueva). Debe incluir un selector de vista arriba con dos opciones: “Tabla” y “Kanban”, donde Kanban está activo.

El diseño debe ser moderno, limpio, minimalista y altamente funcional, estilo Notion / Asana / Monday.

🧩 ESTRUCTURA GENERAL:

Mantener el header existente:
Título: “Procesos y Tareas”
Subtítulo: “Agenda, tareas, proyectos, KPIs y procesos recurrentes”
Mantener tabs principales:
Agenda (activo)
Proyectos
Procesos
KPIs
Organigrama
Debajo de los tabs:
Agregar selector de vista:
Botón “Tabla”
Botón “Kanban” (activo con estilo destacado)

🧱 DISEÑO DEL KANBAN:

Crear un tablero Kanban horizontal con scroll si es necesario.

Columnas:

🟡 Pendiente
🔵 En progreso
🟢 Completado
🔴 Vencido (opcional)

Cada columna debe:

Tener título claro
Mostrar número de tareas
Tener fondo ligeramente diferenciado (muy sutil)
Tener separación clara entre columnas

📦 TARJETAS (TASK CARDS):

Cada tarjeta debe incluir:

Nombre de la tarea (bold)
Responsable (texto pequeño)
Prioridad (badge: Alta / Media / Baja)
Progreso (%) opcional
Fecha de vencimiento

🎨 COLORES (MUY IMPORTANTE):

Seguir el sistema de colores del producto:

🟡 Amarillo (#FFD650 aprox) → acción / tareas
🔵 Azul → información / progreso
🟢 Verde → completado
🔴 Rojo → vencido / alerta

Usar colores suaves, no saturados, con fondos tipo:

badges
bordes
highlights

NO usar colores sólidos agresivos.

✨ INTERACCIONES (VISUALES):

Tarjetas con sombra ligera
Hover en tarjetas (elevación)
Espaciado amplio (UI respirable)
Bordes redondeados (16px aprox)

⚙️ FUNCIONALIDAD VISUAL:

Las tarjetas deben parecer “movibles” (drag & drop)
Mostrar iconos pequeños (clip, usuario, etc.)
Permitir múltiples tarjetas por columna

📐 ESTILO GENERAL:

Diseño tipo dashboard SaaS moderno
Uso de cards
Grid limpio
Tipografía clara
Jerarquía visual fuerte

🚫 EVITAR:

Diseño plano sin jerarquía
Colores saturados
UI pesada o sobrecargada
🔥 BONUS (si quieres subirlo más)

Puedes agregar esto al final:

Incluye una versión mobile responsive donde el Kanban se convierte en scroll vertical por columnas.
